# PRD - 测试任务

## 1. 项目概述

### 项目背景
本项目用于测试工作流系统的基本功能，验证任务分发、角色流转和状态管理机制。

### 项目目标
- 验证工作流任务分发机制
- 验证各角色之间的任务传递
- 验证 TASKS.md 状态更新

## 2. 功能需求

### 2.1 任务分发功能
- 支持任务分发到 product 角色
- 任务包含项目名称和任务描述
- 支持任务状态跟踪

### 2.2 角色流转功能
- product 完成需求分析后自动触发 architect
- architect 完成架构设计后自动触发 developer
- developer 完成功能开发后自动触发 qa
- qa 完成测试后自动触发 writer

### 2.3 状态管理功能
- 任务状态正确流转：待开始 → 处理中 → 已完成
- 当前阶段正确更新
- 下一步角色正确切换

### 2.4 触发机制
- 任务完成后自动触发下一步
- 需要触发标志正确设置

## 3. 验证场景

### 3.1 完整流程验证
- 验证 product → architect → developer → qa → writer 链路
- 验证每个环节的自动触发机制

### 3.2 状态一致性验证
- 验证 TASKS.md 中状态一致性
- 验证当前阶段和下一步角色正确

## 4. 验收标准

- [ ] 任务能够成功分发到 product
- [ ] product 完成后自动触发 architect
- [ ] architect 完成后自动触发 developer
- [ ] developer 完成后自动触发 qa
- [ ] qa 完成后自动触发 writer
- [ ] 每个阶段的状态正确更新
